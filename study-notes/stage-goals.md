# 学习阶段目标

学习体系按“先会用、再理解、再扩展、再贡献”推进。每个阶段都有独立产物和验收标准，避免把文件浏览误当成学习完成。

## 阶段 1：用好 Codex CLI

目标：在本地仓库中受控地完成解释、计划、修改、验证和复盘任务。

完成标准：10 张 Feature Study 卡片均有稳定文档、最小实验、GitHub issue 和可验证结论；能够正确使用范围、权限、配置、扩展和 Git 闭环。

状态：已完成。

## 阶段 2：理解核心源码链路

目标：说明一次用户输入如何进入 session、构造模型请求、执行工具、回填结果和管理上下文。

完成标准：

- 写出从 CLI 到 `RegularTask`、`run_turn`、模型流和工具分发的端到端链路。
- 标出每一段的关键模块、输入/输出和至少一个不变量。
- 解释 approval/sandbox、context compaction、MCP runtime 在链路中的介入点。
- 用源码检索和至少一个已记录的行为实验交叉验证，不把推断写成事实。

状态：已完成；详细记录见 [阶段 2：核心源码链路](stage-2-source-reading.md)。

## 阶段 3：扩展与运行机制

目标：在不扩大默认权限的前提下，理解并实现可维护的 skill、MCP、hook 或 plugin 扩展。

完成标准：

- 为一个重复学习流程选择合适的扩展层级，并说明为什么不选择更重的方案。
- 在隔离范围内完成一个扩展的配置、行为验证和故障路径记录。
- 说明该扩展的权限、生命周期、上下文注入和卸载/回滚边界。

状态：未开始。

## 阶段 4：上游协作与贡献

目标：把学习结果转化为可审阅、可维护的上游贡献能力。

完成标准：

- 能从最新 `main` 创建独立贡献分支，并保持学习分支与上游同步。
- 能为一个真实问题给出复现、根因、最小修复和对应测试。
- 能完成一次本地 review、提交、PR 描述和 review feedback 响应闭环。

状态：未开始。
