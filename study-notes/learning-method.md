# 学习方法探索

这份方法用于学习 Codex 和 agent 系统。目标不是一次性读完所有源码，而是持续产出可验证的理解。

## 总体方法

采用四步循环：

1. 提问题：先写下本轮要回答的一个具体问题。
2. 追链路：从入口开始跟踪真实代码路径。
3. 做实验：用最小实验验证关键行为。
4. 写结论：记录源码依据、实验结果和未解决问题。

每轮学习只聚焦一个主题，避免在多个模块之间发散。

## 主题选择

优先选择能串起完整 agent 链路的问题：

- 用户输入如何进入 Codex CLI？
- 一轮用户消息如何变成模型请求？
- 模型返回 tool call 后，Codex 如何调度工具？
- 命令执行结果如何回填到下一轮模型上下文？
- sandbox、approval policy、exec policy 分别控制什么？
- Codex 如何处理长上下文和 compaction？
- MCP、hooks、skills 分别适合扩展什么能力？

## 阅读策略

先纵向，再横向。

- 纵向：从一个用户动作开始，跟踪入口、核心逻辑、工具执行、结果回传。
- 横向：在理解链路后，再比较相关模块的职责边界。

读源码时优先记录：

- 入口文件和关键函数。
- 数据结构如何流动。
- 哪些地方做权限、状态或错误处理。
- 当前理解依赖了哪些源码路径。

## 实验策略

每个实验只验证一个机制。

推荐实验类型：

- 配置实验：修改最小配置，观察行为变化。
- 工具实验：触发一次命令执行或工具调用，记录输入输出。
- 边界实验：验证权限、沙箱、失败路径或超时行为。
- 对照实验：同一任务在不同配置下分别运行。

实验目录建议：

```text
experiments/
  YYYYMMDD-topic/
    README.md
    commands.md
    observations.md
```

实验 README 至少包含：

- 目的
- 操作步骤
- 观察结果
- 对应源码路径
- 结论或新的问题

## 记录模板

```markdown
## 问题

本轮想回答什么？

## 结论

当前可以确定什么？

## 源码依据

- 路径：
- 关键类型/函数：

## 实验

- 命令：
- 结果：

## 仍然不确定

- 问题 1
- 问题 2

## 下一步

下一轮最小问题是什么？
```

## 复盘标准

一次学习不要求完全理解，但必须留下可继续推进的结果：

- 能说清一个链路。
- 能指出一个模块的职责边界。
- 能复现一个行为。
- 能提出一个更小、更明确的下一个问题。

## 反模式

- 只读目录结构，不追真实执行链路。
- 一次打开太多模块，最后没有结论。
- 只写主观理解，不记录源码路径。
- 做实验但不保存命令和观察结果。
- 在学习分支里改核心逻辑，却没有测试或回滚方案。
