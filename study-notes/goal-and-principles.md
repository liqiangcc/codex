# 学习目标和原则

## 目标

通过这个 fork 系统学习 Codex CLI 和 agent 系统，同时保持能持续合入 `openai/codex` 的最新代码。

重点掌握：

- Codex 从用户输入到模型请求的完整链路。
- Agent loop、tool calling、上下文管理、权限和沙箱机制。
- 本地命令执行、MCP、hooks、skills 等扩展点。
- 如何在个人学习分支中记录理解、做小实验，并保持 `main` 跟随上游。

## 原则

1. `main` 永远只同步 `openai/codex/main`，不放学习笔记和实验代码。
2. 所有学习笔记和实验只放在 `00-study/codex-agent-learning`。
3. 先追踪一条完整执行链路，再横向阅读模块。
4. 每条笔记尽量关联源码路径、commit、命令输出或实验结果。
5. 实验要小而可复现，每个实验只验证一个机制。
6. 每次深入学习前先同步上游，避免基于旧代码形成结论。
7. 区分三类工作：学习记录、个人实验、准备贡献上游。
8. 不修改核心执行、权限或沙箱逻辑，除非已经明确理解风险和验证方式。

## 工作分类

- 学习记录：源码阅读、模块解释、问题清单、术语整理。
- 个人实验：最小复现、行为验证、工具调用观察。
- 上游贡献：bug fix、测试补充、文档改进，单独从最新 `main` 创建贡献分支。

## 判断标准

一次学习任务完成时，至少留下下面一种结果：

- 一个源码链路图或文字说明。
- 一个能复现的实验。
- 一个明确的问题清单。
- 一个可以继续深入的下一个入口。
